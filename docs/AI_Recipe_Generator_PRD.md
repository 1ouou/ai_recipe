# AI 食材魔法师 (Ingredient Wizard) - 产品需求文档 (PRD)

## 1. 项目概述 (Overview)
### 1.1 背景
在快节奏的现代生活中，很多人在做饭时面临“不知道吃什么”或者“冰箱里只剩这几样食材，能做什么”的困扰。传统的菜谱 APP 通常需要用户先想好菜名再去搜索，无法根据现有食材进行灵活的逆向推荐。
### 1.2 目标
打造一款极简的 Web 应用，用户只需输入手头现有的食材，AI 即可智能生成详细的烹饪方案，帮助用户减少食材浪费，激发烹饪灵感。
### 1.3 核心价值
*   **降低决策成本**：不需要想“吃什么”，让食材决定菜谱。
*   **减少浪费**：通过 AI 组合，有效清理冰箱库存。
*   **个性化**：根据用户提供的食材进行精准匹配，而非简单的关键词搜索。

## 2. 用户角色 (User Personas)
*   **独居青年**：下班回家不想点外卖，想用简单的食材快速做顿饭。
*   **家庭主妇/煮夫**：负责一家人的饮食，需要通过现有库存变花样。
*   **烹饪新手**：不知道食材如何搭配，需要详细的步骤指导。

## 3. 功能需求 (Functional Requirements)

### 3.1 核心功能 (MVP)
| 模块 | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **首页** | 食材输入 | 提供一个显眼的输入框，支持逗号分隔输入多个食材（如：鸡蛋, 番茄, 葱）。 | P0 |
| | 预设标签 | 提供常见食材（如：土豆、牛肉、青菜、鸡蛋）的快速点击标签。 | P1 |
| | 生成按钮 | 点击“变出美食”按钮，触发 AI 请求。 | P0 |
| **菜谱列表** | 推荐列表 | AI 返回 1-3 个推荐菜谱，包含：菜名、难度（⭐）、预计耗时、简短描述。 | P0 |
| **菜谱详情** | 基础信息 | 展示菜名、烹饪时长、难度、卡路里预估（可选）。 | P0 |
| | 食材清单 | 分为“主料”（用户输入的）和“辅料”（家中常备如油盐酱醋，需提示用户检查）。 | P0 |
| | 步骤指导 | 结构化的烹饪步骤，语言通俗易懂。 | P0 |
| | 烹饪小贴士 | AI 提供的成功秘诀（如：火候控制、去腥技巧）。 | P1 |

### 3.2 扩展功能 (Future)
*   **口味偏好**：在生成前选择“辣度”、“菜系（中式/西式）”。
*   **图片生成**：使用 DALL-E 或 Stable Diffusion 根据菜谱生成效果图。
*   **保存与分享**：用户登录后可收藏喜欢的菜谱，或生成图片卡片分享给朋友。

## 5. 界面原型说明 (UI Sketches)
### 5.1 首页
*   **Header**: 简洁的 Logo + 项目名称（AI 食材魔法师）。
*   **Hero Section**: 大标题“把冰箱剩下的食材变成美味”，中间是巨大的搜索框/输入框。
*   **Input**: `placeholder="试试输入：鸡蛋, 面粉..."`。
*   **Tags**: 输入框下方放置一行热门食材标签，点击即可添加到输入框。
*   **Action**: 醒目的“开始烹饪”或“生成菜谱”按钮。

### 5.2 详情页
*   **Title**: 菜名（例如：家常番茄炒蛋）。
*   **Tags**: ⏱️ 10分钟 | 🔥 难度：简单。
*   **Ingredients**: 清单列表，勾选框样式（方便用户确认）。
*   **Steps**: 有序列表，步骤清晰，关键动作加粗。

## 6. 技术架构 (Technical Architecture)
*   **前端**: React (Vite) + Ant Design / Tailwind CSS（响应式设计）。
*   **后端**: Node.js (Express/NestJS) 或 Next.js API Routes (BFF层)。
*   **AI 服务**: OpenAI GPT-3.5/4 或 DeepSeek API。
    *   *System Prompt 示例*: "你是一个专业的五星级大厨，用户会给你输入一组食材，你需要推荐 3 道适合家庭烹饪的菜谱，并以 JSON 格式返回，包含菜名、所需食材、步骤..."

## 7. 非功能需求
*   **响应速度**: AI 生成过程可能有延迟（3-10秒），需设计优雅的 Loading 动画（如锅铲翻炒动画）以缓解用户焦虑。
*   **移动端适配**: 优先适配手机浏览器，方便用户在厨房边看手机边操作。
*   **错误处理**: 当 AI 无法根据奇怪的食材生成菜谱时，给予幽默且友好的提示。
